# 朝代数据显示问题修复说明

## 问题描述
用户反馈在朝代大事纪和历史小故事中，选择了朝代后，对应数据没有显示出来。

## 问题原因
1. 在故事渲染逻辑中，添加了容器内容检查（`container.children.length === 0`），导致如果容器已有内容则跳过渲染
2. 在故事筛选逻辑中，只处理了故事容器的显示和隐藏，但没有调用渲染故事的函数
3. 在事件筛选逻辑中，同样只处理了事件容器的显示和隐藏，没有调用渲染事件的函数
4. 页面加载时没有默认渲染数据

## 修复方案
1. **修复故事渲染逻辑**：移除容器内容检查，确保每次切换朝代时都重新渲染内容
   - 文件：`js/main.js`
   - 修改位置：`renderStories` 函数

2. **修复故事筛选逻辑**：在三个地方添加渲染故事的调用
   - 文件：`js/main.js`
   - 修改位置：
     - 朝代按钮点击事件
     - 上一个朝代按钮点击事件
     - 下一个朝代按钮点击事件

3. **修复事件筛选逻辑**：添加渲染事件的调用
   - 文件：`js/main.js`
   - 修改位置：朝代按钮点击事件

4. **添加默认渲染**：在页面加载完成后，默认渲染夏朝的事件和故事
   - 文件：`js/main.js`
   - 修改位置：`DOMContentLoaded` 事件处理函数

## 测试步骤
1. 打开主页面 `index.html`
2. 点击"朝代大事纪"或"历史小故事"选项卡
3. 点击不同的朝代按钮，查看数据是否正确显示
4. 使用上一个/下一个朝代按钮切换，查看数据是否正确显示

## 预期结果
切换朝代时，对应的事件和故事数据应该正确显示在页面上。
