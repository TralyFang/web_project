双击格式化行号更新修复验证

问题：双击格式化后没有更新行数
原问题：双击格式化JSON后，左侧行号区域没有更新显示新的行数
修复方案：在格式化方法中添加行号更新调用
测试用例
测试用例1：压缩JSON格式化
{"name":"test","value":123,"enabled":true,"items":[1,2,3]}
预期：双击格式化后显示多行，行号更新
测试用例2：半格式化JSON
{"users":[{"id":1,"name":"张三"},{"id":2,"name":"李四"}],"metadata":{"total":2,"page":1}}
预期：双击格式化后显示完整格式化，行号更新
验证步骤
  1. 打开JSON解析工具
  2. 输入测试用例1的压缩JSON
  3. 记录当前行号显示（应该显示1行）
  4. 双击输入框进行格式化
  5. 确认左侧行号区域更新为多行
  6. 输入测试用例2的半格式化JSON
  7. 双击格式化
  8. 确认行号正确更新
技术实现细节
修改的文件： app.js
修改位置1：formatJSON() 方法
if (result.success) {
    const formatted = this.parser.formatJson(result.data);
    this.jsonInput.value = formatted;
    this.updateLineNumbers(); // 新增：更新行号显示
    this.parseJSON();
}
修改位置2：basicFormat() 方法
this.jsonInput.value = formatted;
this.updateLineNumbers(); // 新增：更新行号显示
this.parseJSON();

  • ✅ 基础格式化时更新行号 - 完成
